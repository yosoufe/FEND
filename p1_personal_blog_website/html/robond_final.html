Robotics Engineer ND Final Project

Introduction

I started the Robotics Engineer Nanodgree a year ago as a beta tester. So I during the beta testing I finished almost two projects of it. Then later I was looking for an opportunity to finish it. During the COVID-19 crisis Udacity made its nanodegrees free for a month so I used this opportunity to finish my Robotics nanodegree. It was a good experience.

I would like to write about the final project here. The final project was about using different ROS packages to do mapping, localization and navigation of a simulated robot in Gazebo environment. The project can be found here at my github account. The animation above is showing the final results that the robot is navigating to a predefined location in the map, marked with green virtual object. It picks up the object and moves to another location to drop the object virtually. You can find the high quality video of the final project on the YouTube video below.

Requirements

In order to make sure that all the required packages are installed, I used this command:

sudo apt install ros-kinetic-turtlebot-*

To build the project:

cd catkin_ws_home_service_robot catkin_make source devel/setup.sh

The file structure of the project is as follow

├── # Official ROS packages | ├── slam_gmapping # gmapping_demo.launch file │ ├── gmapping │ ├── ... ├── turtlebot # keyboard_teleop.launch file │ ├── turtlebot_teleop │ ├── ... ├── turtlebot_interactions # view_navigation.launch file │ ├── turtlebot_rviz_launchers │ ├── ... ├── turtlebot_simulator # turtlebot_world.launch file │ ├── turtlebot_gazebo │ ├── ... | ├── integration # my launch and world files to use above packages │ ├── launch │ ├── worlds | ├── maps # map files │ ├── ... ├── scripts # shell scripts files │ ├── ... ├──rvizConfigs # rviz configuration files │ ├── ... ├──pick_objects # pick_objects C++ node │ ├── src/pick_objects_node.cpp │ ├── ... ├──add_markers # add_marker C++ node │ ├── src/add_markers_node.cpp │ ├── ... └──

Mapping

Gmapping package is used to create the map of the environment. The created maps can be found in the maps directory. The src/maps/map3 was the best map that I could create. I tried mapping with different world models and parameters for gmapping package and map3 was the best results.

In order to redo the mapping it is enough to do the following commands:

cd catkin_ws_home_service_robot/src/scripts ./test_slam.sh

Now choose the xterm that is running the keyboard_teleop.launch and navigate the robot manually to generate the map. keyboard_teleop.launch is part of turtlebot_teleop package. To learn more, please take a look at here. When all parts of the map are covered, use the following command to save the map:

rosrun map_server map_saver -f outputFile

Localization and Navigation

test_navigation.sh script is a demo of the navigation and localization. To run it:

cd catkin_ws_home_service_robot/src/scripts ./test_navigation.sh

This script is using ROS Navigation stack and Adaptive Monte Carlo Localization (AMCL) for localization under the hood which is being included in catkin_ws_home_service_robot/src/integration/launch/amcl_demo.launch file. It is using Dijkstra's algorithm for navigation which is a variant of the Uniform Cost Search algorithm, while avoiding obstacles on its path.
